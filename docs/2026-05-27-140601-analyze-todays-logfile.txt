⏺ Confirmed: SAPL exposes a standardized public API. Here's the strategy shift.

  The big finding

  Every SAPL-based câmara (most municipalities) exposes a public, paginated JSON
   API:

  GET https://sapl.<municipio>.<uf>.leg.br/api/parlamentares/parlamentar/?format
  =json

  Returns a structured list with exactly the fields you want, no crawling, no
  AI:

  {"nome_completo": "...", "nome_parlamentar": "...",
   "email": "...", "telefone": "...", "telefone_celular": "...",
   "endereco_web": "...", "ativo": true, ...}

  I tested two random instances live (sapl.al.pi.leg.br,
  sapl.joaopessoa.pb.leg.br) — both responded with the same schema. SAPL is
  built by Senado/Interlegis and deployed to thousands of municipalities, with
  the OpenAPI schema at /api/schema/swagger-ui/.

  Recommended pipeline (replaces the current AI crawler for ~80% of cases)

  Tier 1 — Canonical URL directory (one-shot, replaces your input file)
  Download the SIGI master CSV:
  https://sigi.interlegis.leg.br/servicos/casasatendidas/?format=csv (or per-UF,
   e.g. /PR/?format=csv). Columns include casa_legislativa__nome,
  municipio__nome, uf__sigla, email, telefone, url. This gives you the
  authoritative chamber URL and an institutional email for every câmara —
  exactly what's blowing up today with timeouts on guessed .leg.br / .gov.br
  URLs.

  Tier 2 — SAPL API (structured, ~100ms per municipality)
  For chambers running SAPL (detect by sapl.<host> subdomain or by probing the
  URL from the SIGI CSV), hit
  /api/parlamentares/parlamentar/?ativo=true&format=json. Paginate. Zero AI
  cost, zero Playwright.

  Tier 3 — Portal Modelo @@pl-json / @@transparency-json
  For chambers on the Plone-based Portal Modelo, these views return parlamentar
  + email in JSON. Standard endpoint at the site root.

  Tier 4 — Câmara dos Deputados / Senado APIs (out of scope for vereadores, but
  worth knowing): dadosabertos.camara.leg.br and
  legis.senado.leg.br/dadosabertos cover federal layer with full schemas.

  Tier 5 — Targeted HTML scrape (only when tiers 1–3 fail)
  Most câmaras have a predictable /vereadores/, /institucional/contato, or
  /transparencia/dados-abertos page. Skip the Playwright-driven AI navigation
  entirely — fetch with httpx and regex-extract mailto: + neighboring text. The
  current AI navigation is paying ~6–10s per Ollama call × 2–3 hops × 26
  vereadores, which is the bulk of your 25-min runtime.

  Tier 6 — Current AI crawler as last resort. Today it produced 2 hits in 26
  attempts (8%); reserve it for the long tail.

  Why today's run looked so bad

  - Bad input URLs. 12/26 timed out or hit dead-URL cache. Today's input was
  guessed URLs (http://www.riobranco.ac.leg.br/, https://altos.pi.leg.br/).
  SIGI's CSV is the canonical source — use it instead of generating URLs.
  - AI is the wrong tool for this shape of data. When the page has 1 candidate
  email and the AI returns None (politician #1, log line 11:45:48), the issue is
   that institutional emails (e.g. gabinete@camaramunicipal...) ARE the right
  answer for "how do I contact this vereador" but the prompt is asking for a
  personal match. The SAPL API sidesteps this entirely by returning the
  vereador's registered email directly.
  - Playwright 15s timeout is too tight for slow .gov.br infra (3 hard errors
  today). But you mostly don't need a browser — these are static HTML or JSON.

  What this tells us about strategy
